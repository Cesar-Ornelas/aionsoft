# Product And Content Baseline

## Required before launch

- Route inventory and page ownership list
- Final product copy for primary routes
- Metadata title/description for each public route
- Analytics requirement decision and Swetrix project id (if enabled)

## Content rules

- Every primary route should have a single clear user outcome.
- Avoid placeholder copy in production deployments.
- Keep call-to-action text specific and measurable.

## Analytics rules

- Store analytics ids in env vars only.
- Verify no-script fallback tracking path for analytics if required.
- Document analytics endpoint ownership and privacy expectations.
